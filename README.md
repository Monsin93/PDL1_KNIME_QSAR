# PD-L1 Machine Learning Workflow (Supporting Repository)

This repository provides the supporting materials associated with the manuscript:

"Machine-Learning-Guided Discovery and Prioritization of PD-L1 Inhibitors:
An Open-Source KNIME Workflow with Curated Bioactivity Data, Molecular
Fingerprints, and Applicability-Domain Assessment."

The purpose of this repository is to support transparency and reproducibility by
sharing the curated dataset and the executable KNIME workflow used in the study.
All detailed procedures are fully described in the manuscript and Supporting
Information.

## Repository contents

- `PDL1_RFHybrid_github.knwf` — Executable KNIME workflow for the selected model
  (Random Forest with hybrid MACCS + Morgan fingerprints), covering data input,
  outlier removal, fingerprint generation, train/test/external-validation
  splitting, and model training and evaluation.
- `1. ChEMBLdatasetretrivedfromdatabase.csv` — Frozen snapshot of the raw ChEMBL
  query output, 15,504 records (see Data provenance below).
- `2. ChEMBL_curated_dataset.csv` — Curated PD-L1 bioactivity dataset
  (final modeling set, 2,413 compounds).
- `3. PDL1_Data_trainingset.csv`,
  `4. PDL1_Data_testset.csv`,
  `5. PDL1_Data_externalvalidationset.csv` — Train/test/external-validation
  splits used in the study.
- `6. BindingDB_application_dataset_predictedvalues.csv` — Independent BindingDB
  application set with predicted values (cross-database analysis).
- `FeturesSelectionandSHAP_analysis.ipynb` and `ForRunFeatureSelection_colab/` —
  Python/Colab notebooks for feature selection and SHAP analysis.
- Supporting Information (KNIME workflow diagrams for each pipeline step) is
  provided in the Supplementary file accompanying the manuscript.

## Data provenance and reproducibility

The primary modeling data originate from ChEMBL (target ID: CHEMBL4523993;
accessed 16 October 2025), retrieved using the TeachOpenCADD-based acquisition
step described in the manuscript (Methods 2.2) and Supplementary Figures S1–S4.

To guarantee exact reproducibility, the raw query output is provided as a fixed
snapshot rather than re-queried live. ChEMBL is updated continuously, so a live
query run on a later date may return a different number of records and would not
reproduce the published results. The file `data/Data_CHEMBL15504.csv` contains
the 15,504 raw bioactivity records retrieved on the access date. Applying the
documented curation and filtering steps (duplicate removal, Alpha/HTRF assay
restriction, unit normalization to nM, pIC50 conversion, and outlier screening)
yields the final modeling set of 2,413 unique compounds used for model
development.

## Running the workflow

1. Requires KNIME Analytics Platform v5.2.1 (or later) with the RDKit KNIME
   extension installed.
2. The input node reads the raw dataset
   (`1. Data_CHEMBL15504.csv`) via a path relative to the
   workflow; keep the file in place and the workflow will locate it
   automatically.
3. Reset and execute the workflow to reproduce the train/test/external-validation
   splits and the Random Forest model.

## Trained model

A pre-trained model file is intentionally not included. The selected Random
Forest model is fully regenerated by running the workflow on the frozen dataset
with the fixed random seeds specified in the node configuration. Providing a
serialized model object is therefore unnecessary and would tie reuse to a
specific KNIME build.

## Scope of the released workflow

The released workflow corresponds to the selected model reported in the
manuscript (Random Forest, hybrid MACCS + Morgan). The complete architecture and
hyperparameters of all benchmarked algorithms, including the Keras
deep-neural-network baseline, are documented in the manuscript (Table 2) and
Supporting Information.
